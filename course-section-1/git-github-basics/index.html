Repositories & Commits

A repository is a collection of code.

A commit in Git is a snapshot of the repository.

Staging

  • git add <file-name> – stage a specific file
  • git add . – stage all files in the repository

Committing

  • git commit -m "Your commit message" – commit with a message
  • git commit – commit without a message; Git will open an editor to type one

Checking Status

  • git status – shows modified, staged, and untracked files

Viewing Commit History

  • git log – shows commit history with commit IDs, authors, and messages

Pushing Changes

  • git push – pushes commits to the default remote and branch

Checking Out Commits

  • git checkout <commit-id> – moves to a specific commit (detached HEAD)
  • git checkout <branch-name> – return to a branch

Reverting Commits

  • git revert <commit-id> – creates a new commit that reverses the changes of the given commit

Resetting Commits

  • git reset --soft <commit-id> – keep changes staged
  • git reset --mixed <commit-id> – unstage changes but keep them in working directory
  • git reset --hard <commit-id> – discard all changes

Ignoring Files

  • Create a .gitignore file in the repository root.
  • Add patterns to ignore (examples):
    • node_modules/ – ignore dependencies
    • *.log – ignore all log files
    • dist/ – ignore build output
    • secret.txt – ignore a specific file

Working with Branches

Branches let you work on features or fixes independently from the main codebase:

  • git branch – list all branches in the repository
  • git branch <branch-name> – create a new branch
  • git checkout <branch-name> – switch to a branch
  • git checkout -b <branch-name> – create and switch to a new branch in one step
  • git merge <branch-name> – merge another branch into the current branch
  • git branch -d <branch-name> – delete a branch (safe, if merged)
  • git branch -D <branch-name> – force delete a branch (even if not merged)
