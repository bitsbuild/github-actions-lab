Repositories & Commits

A repository is a collection of code.

A commit in Git is a snapshot of the repository.

Staging

Staging in Git is marking files to be included in the next commit. Commands:

  • git add <file-name> – stage a specific file
  • git add . – stage all files in the repository

Committing

Creating a commit saves the staged changes:

  • git commit -m "Your commit message" – commit with a message
  • git commit – commit without a message; Git will open an editor to type one

Checking Status

Check the state of your working directory and staging area:

  • git status – shows modified, staged, and untracked files

Viewing Commit History

See past commits:

  • git log – shows commit history with commit IDs, authors, and messages

Pushing Changes

Send your local commits to a remote repository:

  • git push – pushes commits to the default remote and branch

Checking Out Commits

To view a previous state of the repository:

  • git checkout <commit-id> – moves to a specific commit (detached HEAD)
  • To return to the latest commit on a branch: git checkout <branch-name>

Reverting Commits

If you want to undo a commit without changing history:

  • git revert <commit-id> – creates a new commit that reverses the changes of the given commit

Resetting Commits

If you want to undo commits and optionally remove changes:

  • git reset --soft <commit-id> – moves branch pointer but keeps changes staged
  • git reset --mixed <commit-id> – moves branch pointer and unstages changes (default)
  • git reset --hard <commit-id> – moves branch pointer and discards all changes
